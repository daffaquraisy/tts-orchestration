fig.2 async request lifecycle
01
POST /synthesize
Client sends text payload. Pydantic validates SynthesisRequest. Empty text returns 422 before hitting business logic.
02
Enqueue to Redis
uuid4 job_id generated. Status written to Redis as queued. Job payload pushed to RQ queue. API returns 202 immediately non-blocking.
03
Worker dequeues
RQ worker picks up job from Redis. Updates status to processing. Runs in a separate container process.
04
Text analysis
Rule-based checks: presence of ?, !, formal keywords (dear, formally, sincerely). Computes word count and avg sentence length. Returns TextAnalysis model.
05
Voice routing
Iterates rules in JSON order first match wins. Falls back to default_rule if no condition matches. Returns VoiceRule with voice_id, rate, volume, pitch.
06
pyttsx3 synthesis
Applies rate, volume, voice to engine. Calls save_to_file to /tmp/job_id.wav. Cleans up partial file on failure and raises SynthesisError.
07
Write final status
On success: status=done, audio_path written to Redis. On failure: status=failed, error message written.
08
Client polls GET /status/job_id
API reads from Redis and returns SynthesisJob. Unknown job_id returns 404.
